Univariate Guided Sparse Regression

Notes from a talk on Unilasso, an adaptive regularization method that adjusts penalty strength based on coefficient magnitude. The core motivation is that LASSO applies a uniform \(\ell_1\) penalty across all coefficients, which over-shrinks large ones and produces biased estimates for strong predictors.

Background: LASSO and its limitations

The standard LASSO solves:

LASSO objective
\[ \hat{\beta} = \arg\min_{\beta} \left\{ \frac{1}{2n} \| y - X\beta \|_2^2 + \lambda \sum_{j} |\beta_j| \right\} \]

The uniform \(\lambda |\beta_j|\) penalty is the source of two problems that Unilasso targets. First, large true coefficients are shrunk by the same amount as small ones, introducing bias that does not vanish as \(n \to \infty\) unless \(\lambda \to 0\). Second, the selection boundary is the same for all predictors regardless of their signal strength, so weakly relevant predictors are discarded at the same threshold that also removes noise.

The Unilasso formulation

Unilasso replaces the uniform \(\ell_1\) penalty with a separable adaptive penalty \(P_\lambda(\beta) = \sum_j \rho_\lambda(\beta_j)\), where:

Unilasso penalty
\[ \rho_\lambda(\beta) = \lambda \frac{|\beta|}{1 + \gamma |\beta|} \]

The parameter \(\gamma > 0\) controls how quickly the penalty saturates for large \(|\beta|\). The key behaviors:

  • Small \(|\beta|\) \(\rho_\lambda(\beta) \approx \lambda|\beta|\) — behaves like LASSO, aggressively shrinking small coefficients toward zero.
  • Large \(|\beta|\) \(\rho_\lambda(\beta) \to \lambda/\gamma\) — penalty saturates, so strong predictors experience diminishing penalization.
  • Gradient \(\rho'_\lambda(\beta) = \lambda / (1 + \gamma|\beta|)^2\) — decreasing in \(|\beta|\), which is the mechanism of bias reduction.

The gradient of the full penalty used in coordinate descent is:

Penalized gradient
\[ \frac{\partial}{\partial \beta_j} P_\lambda(\beta) = \frac{\lambda \operatorname{sign}(\beta_j)}{(1 + \gamma |\beta_j|)^2} \]

Since \(\rho_\lambda\) is nonconvex, the overall objective is nonconvex. Coordinate descent converges to a stationary point rather than a global minimum, which is standard for nonconvex penalized regression. In practice this is handled the same way as SCAD and MCP.

Theoretical guarantees

The talk establishes the following properties under standard sparsity and design conditions (bounded eigenvalues of \(X^TX/n\), sub-Gaussian noise):

  • Oracle property With appropriate \(\lambda_n\), Unilasso asymptotically selects the true support and the estimates on the support converge at the parametric rate, as if the true support were known in advance.
  • Selection consistency \(P(\hat{S} = S^*) \to 1\) where \(S^* = \{j : \beta^*_j \neq 0\}\). Unlike LASSO, this holds without requiring the irrepresentability condition.
  • Estimation bias For \(\beta_j^* \neq 0\), the bias of \(\hat\beta_j\) decays as \(O(\lambda / (1 + \gamma|\beta_j^*|)^2)\), which goes to zero for fixed nonzero \(\beta_j^*\) as \(\lambda \to 0\). LASSO bias is \(O(\lambda)\) regardless of coefficient size.
  • Computational complexity Coordinate descent updates are \(O(n)\) per coordinate (same as LASSO), so the method scales to high-dimensional problems.

Comparison with related methods

Method Penalty Key limitation Unilasso vs.
LASSO \(\lambda|\beta|\) Uniform shrinkage, biased for large \(\beta\), requires irrepresentability for selection consistency Unilasso reduces bias via saturating penalty; oracle property holds without irrepresentability
Elastic Net \(\lambda_1|\beta| + \lambda_2\beta^2\) Handles correlated features but does not resolve over-shrinkage of large coefficients Unilasso adaptively reduces penalization for large \(\beta\); one fewer hyperparameter
SCAD Piecewise nonconvex, flat beyond threshold Achieves oracle property but requires tuning of two parameters (\(\lambda\), \(a\)); less interpretable penalty shape Similar oracle property; Unilasso penalty \(\rho_\lambda\) has a single additional parameter \(\gamma\) with clear interpretation
MCP Minimax concave, linear then flat Good bias-variance tradeoff but less smooth penalty derivative at transition Unilasso penalty is smooth everywhere; gradient-based optimization cleaner
Adaptive LASSO \(\sum_j w_j |\beta_j|\) with data-dependent weights Requires a first-stage estimator to set weights; oracle property depends on quality of first stage Unilasso achieves adaptivity within a single optimization; no first-stage estimator needed

What makes this interesting

The core insight is that the oracle property for LASSO requires conditions (irrepresentability, restricted eigenvalue, or similar) that rule out correlated predictors in practice. Nonconvex penalties like SCAD and MCP have been known to circumvent these conditions for a while. What Unilasso adds is a specific functional form, \(\lambda|\beta|/(1 + \gamma|\beta|)\), that has a clear interpretation as a smoothly saturating penalty: it looks like \(\ell_1\) near zero and like a constant for large values. This makes the bias reduction mechanism transparent in a way that SCAD's piecewise definition does not.

The empirical claim that Unilasso requires less tuning than SCAD or MCP is worth watching. In practice, cross-validation over \(\gamma\) is still needed, so the claim is more about stability of performance across values of \(\gamma\) than about truly requiring fewer hyperparameters.
